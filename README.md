# 📦 3-Tier Containerized Web Application in AWS EC2 (Learning Project)

This is a foundational DevOps practice project where I’m learning real-world deployment concepts by building a simple 3-tier containerized application.

The goal is not complexity, but understanding how modern applications are structured, secured, and deployed.

---

## 🏗 Architecture

```
Client → Nginx (Reverse Proxy) → Frontend / Backend → PostgreSQL
```

```bash
Docker-Project/
│
├── backend/               # Node.js backend API
│   ├── Dockerfile
│   ├── package.json
│   └── server.js
│
├── frontend/              # Frontend static site
│   ├── Dockerfile
│   └── index.html
│
├── nginx/                 # Reverse proxy + SSL configuration
│   └── (nginx.conf)
│
├── init.sql               # PostgreSQL initialization script
├── docker-compose.yml     # Orchestration for all services
├── .env                   # Environment variables (not committed)
└── README.md
```
---

### 1️⃣ Frontend
- Static site
- Served via Nginx
- Runs in its own container

### 2️⃣ Backend
- Node.js API
- Runs in a separate container
- Exposes REST endpoints
- Connects to PostgreSQL

### 3️⃣ Database
- PostgreSQL 15-alpine
- Uses Docker volumes for persistent storage

---

## 🔧 Infrastructure Components

- **Reverse Proxy:** Nginx  
- **SSL:** Let’s Encrypt (Certbot)  
- **DNS:** Cloudflare (DNS-Only mode required)  
- **Orchestration:** Docker Compose  
- **Persistence:** Docker Volumes

## ⚠️ Required Setup

### 1. Create `.env` File

Create a `.env` file in the project root:

```bash
DB_HOST=database
DB_USER=your_db_user
DB_PASSWORD=your_db_password
DB_NAME=your_db_name
```

Do not commit this file.

---

### 2. Use Your Own Domain

A real domain is required for:

- SSL certificate generation  
- HTTPS access  
- DNS validation  

---

### 3. Update Nginx Configuration Before Running Containers

Before running `docker compose up`, update the `nginx.conf` file with your domain and SSL configuration.

Replace the values of `server_name` with your actual domain and configure HTTPS using the certificates generated by Certbot.

Example `nginx.conf`:

```bash
server {
    listen 80;
    server_name <yourdomain.com> <www.yourdomain.com>;

    location / {
        proxy_pass http://frontend:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}

server {
    listen 443 ssl;
    server_name <yourdomain.com> <www.yourdomain.com>;

    ssl_certificate /etc/letsencrypt/live/<yourdomain.com>/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/<yourdomain.com>/privkey.pem;

    location / {
        proxy_pass http://frontend:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

Make sure to replace:

```bash
<yourdomain.com>
```
- For example `www.example.com` in place of `<yourdomain.com>`
with your real domain name.

## 🔐 Install Certbot (Ubuntu)

Install Certbot on your server to generate SSL certificates from Let's Encrypt.

```bash
sudo apt update
sudo apt install certbot -y
```

---

## 🔐 Generate SSL Certificate

Before running this command:

- Port **80** must be open on the server.
- Your domain must point to the **public IP of the server**.
- If using Cloudflare, the DNS record must be set to **DNS Only (grey cloud)**.

Run:

```bash
sudo certbot certonly --standalone \
-d yourdomain.com \
--non-interactive \
--agree-tos \
-m your-email@example.com
```

Certificates will be generated in:

```bash
/etc/letsencrypt/live/yourdomain.com/
```

---

## 📦 Mount SSL Certificates into the Nginx Container

To allow the Nginx container to use the certificates, mount the certificate directory as a Docker volume.

Example snippet in `docker-compose.yml`:

```bash
services:
  nginx:
    image: nginx:latest
    volumes:
      - ./nginx.conf:/etc/nginx/conf.d/default.conf
      - /etc/letsencrypt:/etc/letsencrypt
    ports:
      - "80:80"
      - "443:443"
```

This ensures:

- SSL certificates remain **persistent**
- The container can access certificates generated on the host machine

---

## 🚀 Start the Application

Once the certificates are generated and the Nginx configuration is updated, start the containers:

```bash
docker compose up -d
```

---

## 🔁 Renew SSL Certificates

Certificates from Let's Encrypt expire every **90 days**.

To renew them manually:

```bash
sudo certbot renew
```

You can also automate renewal using a cron job:

```bash
0 3 * * * certbot renew --quiet
```
